# Write User Story

Generate a user story following Agile best practices for project features.

## Format

**As a** [type of user]  
**I want** [an action]  
**So that** [a benefit/value]

## Acceptance Criteria

- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Technical Requirements

- [ ] Requirement 1
- [ ] Requirement 2

## Implementation Notes

- Technical approach
- Dependencies
- Configuration needed

## Testing Requirements

- Unit tests
- Integration tests
- Manual test scenarios

## Definition of Done

- [ ] Code implemented and reviewed
- [ ] Unit tests written and passing
- [ ] Integration tests passing
- [ ] Documentation updated
- [ ] Configuration documented
- [ ] No sensitive data in logs
- [ ] Performance requirements met

## Guidelines
- Focus on business value
- Include security considerations
- Consider integration requirements (AWS, external services, etc.)
- Reference relevant domain concepts for the project
